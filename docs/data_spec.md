# Спецификация данных (v1.0)

Этот документ определяет формат хранения заметок и способ взаимодействия между Core и UI.

## 1. Хранение файлов
- Все заметки хранятся в директории `~/notes/` (по умолчанию) или в папке, указанной в конфиге.
- Формат файлов: `.md` (Markdown).
- Имя файла может быть любым, но рекомендуется `YYYY-MM-DD-slug.md`.

## 2. Формат YAML Frontmatter
Каждый файл **обязан** содержать блок YAML в начале.

### Обязательные поля:
- `date`: Дата события в формате `YYYY-MM-DD`.

### Опциональные поля:
- `title`: Заголовок события (если отсутствует, используется имя файла).
- `tags`: Список тегов (например, `[work, university]`).
- `status`: Одно из значений: `todo`, `in-progress`, `done`.

### Пример файла:
---
date: 2023-12-01
title: "Защита MVP"
status: todo
tags: [project, study]
---
